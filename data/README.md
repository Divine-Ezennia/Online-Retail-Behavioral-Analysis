# 1ï¸âƒ£ RAW DATA LAYER (`/raw`)

## Purpose
Stores the original dataset exactly as obtained from its source.

This layer remains:
- Immutable
- Unaltered
- Audit-traceable

## Source Provenance

- **Dataset:** UCI Online Retail Dataset  
- **Source:** UCI Machine Learning Repository  
- **Nature:** Public transactional e-commerce dataset  
- **Volume:** 541,000+ records  

The raw file is intentionally excluded from version control due to size.

## Governance Policy

- No transformation occurs in this layer.
- No derived files are stored here.
- The dataset can be re-downloaded from the official UCI repository.

See: [README.md](./raw/README.md) for full documentation.

# ðŸ“Š Data Engineering Philosophy

This repository does **not** treat data as a static analysis input.

Instead, it enforces:

- Layered transformation logic
- Script-based state generation
- Modular checkpoint artifacts
- Deterministic rebuilding capability

The `.rds` architecture ensures that:

- Each script can run independently after its dependency is satisfied
- Debugging is isolated to transformation stage
- Visualization consumes stable analytical outputs

# â™»ï¸ Reproducibility Protocol

To fully rebuild the data layer:

1. Clone repository
2. Download UCI Online Retail dataset into `/data/raw/`
3. Install required R packages
4. Execute scripts sequentially from `01` through `11`
5. `.rds` artifacts will populate `/data/processed/`
6. Visualization outputs will populate `/visualizations/`

No manual data editing is required.
