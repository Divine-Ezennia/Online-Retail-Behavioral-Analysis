# 📦 Processed Data Layer

This directory contains all intermediate and final analytical artifacts generated by the modular R pipeline located in `/scripts`.  

These files are **derived outputs**, not source data.

They are programmatically generated using `saveRDS()` and consumed downstream via `readRDS()` across scripts 02 → 11.

---

## 🏗️ Architectural Role

The `processed/` directory represents the **transformation layer** within the project’s analytical architecture.

It stores:

- Cleaned datasets
- Enriched datasets
- Aggregated metrics
- Cross-dimensional intelligence outputs
- Segment intelligence artifacts

These artifacts function as **state checkpoints**, allowing:

- Modular execution
- Reproducibility
- Object traceability
- Efficient downstream visualization rendering

---

## 🔄 Data Flow Dependency

The processing hierarchy follows strict sequential execution:

```
01_data_loading.R

02_data_cleaning.R

04_foundational_metrics.R
```

Each script produces `.rds` artifacts consumed by subsequent stages.

## 📂 Artifact Categories

Artifacts are grouped logically by analytical domain:

### 1️⃣ Enriched Core Dataset
- `online_retail_enriched.rds`

#### 🔎 Derived Metadata: Product Label Mapping

Within `03_data_enrichment.R`, a deterministic mapping table is programmatically generated:

- `ShortProductLabel`
- `Description`

This mapping ensures:

- Consistent categorical labeling across all visual outputs
- Readable axis formatting in high-density charts
- Elimination of overly long product descriptions

The mapping table is not stored as a standalone `.rds` artifact because it can be reproducibly regenerated from `online_retail_enriched.rds`.

This design enforces:

- Zero redundancy
- Deterministic rebuild capability
- Clean artifact governance

### 2️⃣ Foundational Metrics
- Unique products and order-based aggregations

### 3️⃣ Product Intelligence
- Top/bottom products by:
  - Total Quantity
  - Average Quantity
  - Total Revenue
  - Average Revenue
  - Order Frequency

### 4️⃣ Temporal Intelligence
- Hourly
- Daily
- WeekPart
- Monthly

Each dimension includes:
- Total metrics
- Average metrics
- Order-line metrics
- Distinct order metrics (where applicable)

### 5️⃣ Geographic Intelligence
- Country-level quantity, revenue, and order metrics

### 6️⃣ Cross-Dimensional Intelligence

### 7️⃣ RFM & Segment Intelligence

## ⚠️ Version Control Policy

This directory is excluded from Git tracking via:

```
data/processed/*.rds
```

### Rationale

- `.rds` files are deterministic outputs
- They can be regenerated by executing scripts in order
- Prevents repository bloat
- Maintains clean commit history
- Enforces pipeline reproducibility

---

## 🔁 Reproducibility Protocol

To regenerate this directory:

1. Clone repository
2. Ensure required R packages are installed
3. Execute scripts sequentially from `01` to `11`
4. `.rds` artifacts will automatically populate `data/processed/`

No manual file manipulation is required.
